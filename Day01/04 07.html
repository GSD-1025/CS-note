Day01:웹 개발 환경 이해하기

작성일: 04.07
작성자: 강성대



●웹

-브라우저로 서비스를 요청과 처리해주는 환경으로 클라이언트가 http 프로토콜(웹 서버와 웹 클라이언트 간의 통신 규약)로 요청(request)을 보내면 웹서버가 이를 처리하고 응답(response)한 후에 연결을 종료한다.

●쿠키, 세션

-http 프로토콜은 연결을 끊는 순간 상태 정보를 유지하지 않는 무상태성(stateless)을 특징으로 가지고 있으며 이러한 약점을 보완하기 위한 방법이 쿠키와 세션이다. 쿠키는 클라이언트에 정보를 저장하며, 세션은 서버가 일정 시간 동안 같은 사용자가 보내는 요청을 하나의 상태로 보고 이를 유지한다.

●html 문서

-웹 서버는 클라이언트로부터 http 프로토콜로 요청을 받았을 때 html 문서로 응답하고 클라이언트는 응답 받은 문서를 렌더링해서 보여준다.

●정적문서, 동적문서

-html 문서는 기본적으로 정적 문서이며 화면이 바뀌지 않고 요청을 받을 경우 전체를 새로 받아서 렌더링한다. 동적 문서는 Javascript 또는 Ajax를 통해서 요청을 받을 경우 부분만 새로 받아서 렌더링한다.
